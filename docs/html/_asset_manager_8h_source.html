8// Asset Management Module
9//-----------------------------------------------
10// ................................................................................
11// All Register__ methods load and store the resource internally
12// The returned handle is used by Get__ methods to retrieve the resource again
13// Per default all methods are made to load from memory to work with the asset image
14// Textures are automatically stitched into the default atlas or a specified one
15// Try to group all textures that are drawn together into the same atlas
16// Uses a handle system, which means you get a number that identifies this resoure
17// However this handle is only unique for its type, so dont use a handle from a spritesheet for a texture or a sound
18// In practice this doesnt mean anything as you organize handles anyway
19// This handle system is very fast - all items from a resource type (Sound, TextureRegion) are stored in a contigous vector
20// The handles are direct indices for lookups in those vectors O(1)
21// All methods return handle::null if there are errors!
22// ................................................................................
